‚ö° git-upload.ps1 - Main Upload Script

What it does:

  • Automatically initializes Git repository if needed
  • Adds GitHub remote using your GitHub username
  • Stages all changes
  • Commits with custom or auto-generated message
  • Pushes to GitHub with conflict resolution
  • Shows repository URL when complete

How to use:

Initial Upload (First Time):

.\git-upload.ps1 -Initial

Creates detailed initial commit with project features listed.

Regular Updates with Custom Message:

.\git-upload.ps1 -Message "Fixed mobile responsiveness issues"

Use descriptive messages about what you changed.

Quick Update (Auto Message):

.\git-upload.ps1

Automatically generates timestamp-based commit message.

When to use:

  • After coding sessions: Upload your changes
  • Before major changes: Create backup point
  • Daily/Weekly: Keep repository updated
  • Before deployment: Ensure code is backed up
Smart Features:
  • Skips upload if no changes detected
  • Handles merge conflicts automatically
  • Shows progress with colored output
  • Displays final repository URL

üîß setup-git.ps1 - Setup Helper

What it does:

  • Initializes Git repository
  • Creates initial commit
  • Provides instructions for adding GitHub remote

When to use:

  • First time only: When setting up Git manually
  • Alternative to git-upload.ps1: If you prefer manual control
Note: The git-upload.ps1 script handles setup automatically, so this script is optional.

üíª install-gh-cli.ps1 - GitHub CLI Installer

What it does:

  • Tries multiple installation methods (winget, manual download)
  • Downloads and installs GitHub CLI
  • Provides post-installation instructions

When to use:

  • Prerequisites: Before using git-upload.ps1
  • One time only: GitHub CLI installation

After installation:

gh auth login gh repo create h-dcn --public --source=. --remote=origin --push

üî® fix-build.ps1 - Build Fixer

What it does:

  • Cleans previous build and cache
  • Fixes npm vulnerabilities
  • Reinstalls dependencies
  • Creates fresh production build

When to use:

  • Chunk loading errors: When S3 deployment fails to load
  • Build corruption: When build seems broken
  • Before deployment: Ensure clean build

üìù Typical Workflow

1. First Time Setup:

# Install GitHub CLI (if needed) .\install-gh-cli.ps1 # Login to GitHub gh auth login # Initial upload .\git-upload.ps1 -Initial

2. Daily Development:

# Make code changes... # Upload with descriptive message .\git-upload.ps1 -Message "Added mobile responsive tables" # Or quick upload .\git-upload.ps1

3. Before Deployment:

# Fix any build issues .\fix-build.ps1 # Upload the fixes .\git-upload.ps1 -Message "Fixed build for deployment" # Deploy to S3 .\deploy.ps1

üö® Troubleshooting

Common Issues:

Error Solution
"gh not found" Run .\install-gh-cli.ps1 and restart PowerShell
"git not found" Install Git from git-scm.com
"remote repository not found" Create h-dcn repository on GitHub first
"authentication failed" Run gh auth login to authenticate
"merge conflicts" Script handles automatically, but check output

üí° Best Practices

  • Commit Often: Upload changes frequently to avoid losing work
  • Descriptive Messages: Use meaningful commit messages
  • Before Major Changes: Create backup point with upload
  • Test Before Upload: Ensure code works before committing
  • Clean Builds: Use fix-build.ps1 before important deployments
